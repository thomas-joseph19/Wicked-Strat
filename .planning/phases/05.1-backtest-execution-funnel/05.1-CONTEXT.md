## Root cause (locked)

1. **`to_trade_result()`** requires `state == CLOSED` **and** non-empty `fills` (`src/position.py`).
2. **`run_session_backtest`** iterated all bars; if the position stayed **`OPEN_FULL` / `OPEN_PARTIAL`** through the **last row** of `bars_df` (e.g. last timestamp **before** 15:45 ET so `bar_is_eod` never ran, or path-dependent), simulation ended **open** → **`tr` = None** despite ENTRY fill.
3. **`PENDING_FILL` + first eligible bar is EOD (≥15:45)** still yields CLOSED **without** fills → `tr` None (by design: no trade); this can account for **some** misses but not the dominant “entered then lost” pattern.

**Mitigation (implemented in code):** After the bar loop in `run_session_backtest`, if state is still open with at least one fill, call `Position.force_end_of_session_flat` on the **last** bar so the backtest always produces a closed position when an entry occurred. Aligns with M1 need for a **complete trade log** when the session stream ends before an explicit EOD bar triggers the normal path.

**Spec note:** Real sessions include 15:45+ bars; this flush mainly guards **truncated slices**, **last bar &lt; 15:45**, and any future bar-set filters. Exit is still tagged **EOD** at last close.
