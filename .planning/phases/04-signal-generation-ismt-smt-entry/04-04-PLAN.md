<interfaces>
**Approach (D-08 long):** At bar `i`, require:
1. `low[i] <= LVN.high + 3 * tick_size`
2. `close[i-1] > LVN.high`, `close[i-2] > LVN.high`, `close[i-3] > LVN.high` (strict)

**Short (D-09):** `high[i] >= LVN.low - 3*tick` and prior 3 closes **all** `< LVN.low`.

**3-Step long trigger (ENTRY-01):** `close[i] > LVN.high` **and** `close[i] > open[i]` (bullish candle).

**Structural:** `get_structural_confirmation("BULLISH", i, ismt_list, smt_list)` is not `None`.

**SP gate (ENTRY-01):** ∃ SP zone with `respected_overnight is True` and `sp.low > entry_price` (use **`entry_price = close[i]`** for validation at signal time).

**Aggressive long (ENTRY-03):** `WindowConfig.AGGRESSIVE_START <= t <= WindowConfig.AGGRESSIVE_END` on bar timestamp **America/New_York**. Proximity: `min(abs(close - lvn.low), abs(close - lvn.high)) <= 3 * tick_size`. Bias **`BULLISH`** (use same enum/string as `tpo.py` will provide — tests stub `session_bias: str`). Trigger: `close > LVN.high` and bullish candle. **Do not** require ISMT/SMT. **`position_size_scale = 0.5`** on `TradeSetup`.

**ENTRY-05 (numeric):** Let `risk = abs(entry - stop)`, `reward = abs(tp1 - entry)` for long (mirror short). Require `reward / risk >= 1.5`. Stop distance in ticks: `risk / tick_size` ∈ `[4, 1.5 * atr20 / tick_size]` (use **price** distance `<= 1.5 * atr20[i]`). **SL prices** for this check: **SL-01/02** formulas: Long `stop = LVN.low - 0.5 * ATR_5` snapped to tick grid; Short `stop = LVN.high + 0.5 * ATR_5` snapped. Use **`atr5[i]`**, **`atr20[i]`** from test fixtures.

**Time:** Reject if `bar_time >= time(15, 45)` (ENTRY-05 “before 3:45 PM ET” — interpret as **no new setups** at or after 15:45 bar labeled time).

**Session counters (D-13):** `setups_emitted_this_session < 3` before returning a new `TradeSetup`.

**D-14 (two-step lifecycle — aligns with 04-CONTEXT D-14 “results in a trade”):**

1. **Do not** treat “Phase 4 returned a `TradeSetup`” as sufficient to suppress aggressive reuse of that LVN for the session. Emitting or returning a setup **without** a subsequent open must **not** add the LVN to `aggressive_lvn_suppressed_for_session`.

2. **Phase 4 (`src/entry.py`):** When an **Aggressive Ledge** candidate **passes** `validate_pre_entry` (ENTRY-05) — i.e. the setup is **accepted** for execution — add a stable LVN key (same formula: e.g. `round(lvn.midpoint / tick_size) * tick_size` or documented `lvn_id: str` on `TradeSetup`) to **`aggressive_lvn_pending_suppression: set`** on the session/state object passed into evaluators. Pending = “accepted aggressive setup, position not yet opened.”

3. **Phase 5 hook (`src/position.py`):** Implement or stub **`enter_from_setup(setup: TradeSetup, ...)`** such that when it **commits** opening a position (first fill / position established — same commit point for all callers), and `setup.setup_type` indicates **Aggressive Ledge**, it calls **`notify_aggressive_trade_completed(lvn_id: str)`** defined in **`src/entry.py`** (single canonical place). That function **moves** the key from `aggressive_lvn_pending_suppression` (if present) into **`aggressive_lvn_suppressed_for_session: set`**.

4. **Aggressive evaluation skip rule:** Before evaluating aggressive on an LVN, skip only if its key is in **`aggressive_lvn_suppressed_for_session`**. That set is updated **only** by **`notify_aggressive_trade_completed`** after **`enter_from_setup`** commits the open. **`aggressive_lvn_pending_suppression`** records ENTRY-05–accepted aggressives for bookkeeping and tests; it does **not** by itself suppress further aggressive evaluation.

5. **Clarification:** 3-Step setups do not use this aggressive suppression path; only aggressive ledge completions feed `notify_aggressive_trade_completed`.
