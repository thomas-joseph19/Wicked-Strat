**Hard stop (TP-05):** True iff `[body_lo, body_hi]` ⊆ `[lvn.low, lvn.high]` (inclusive bounds; use tick tolerance only if existing codebase already uses epsilon — otherwise exact float compare with documented assumption).

**EOD bar:** Bar whose **ET** time satisfies `time >= WindowConfig.EOD_CUTOFF` (15:45) — match **`entry.validate_pre_entry`** convention (`bar_time >= time(15, 45)`). Use same timezone as bar index (**America/New_York**).

**Per-bar order (locked for this plan):**
1. If `state in {OPEN_FULL, OPEN_PARTIAL}` and bar is **EOD** → one **EOD** fill for **remaining** contracts at **`close_nq`** → **CLOSED** (D-08).
2. Else if `state in {OPEN_FULL, OPEN_PARTIAL}` and **hard stop** → one **HARD_STOP_LVN** fill at **`close_nq`** → **CLOSED** (D-07).
3. Else run existing **TP1 / SL / TP2 / breakeven** logic from **05-01** unchanged.

**Interaction with D-05:** On a bar where both TP1 and SL would trigger, **TP1 still wins** because step 3 runs after step 2; hard stop/EOD short-circuit only when applicable.
